TRUST PLATFORM for CryptoAuthentication® devices

The Trust Platform for CryptoAuthentication™ is a combination of multiple elements to simplify the addition of security to any embedded design.

It combines: A unique secure flow to easily protype various security use cases leveraging Microchip CryptoAuthentication™ device family (ATECC608A / ATSHA204A and future devices) and an optimized secure provisioning flow starting at very small minimum quantities: starting at 10 units!

This platform includes Hardware protyping tools along with a unique design suite to accelerate the prototyping of the most common use cases.

The Trust Platform is split in 3 sub-families: Trust&GO, TrustFLEX and TrustCUSTOM which allow different level of flexibility in the customization and provisioning services. Start your journey with Trust&GO and navigate to TrustFLEX / TrustCUSTOM for the highest level flexibility.

METHODOLOGY

To help your design journey, Microchip is offering this Use-case Tool. It is a step by step guide for application developers to go from concept to production. Starting from identifying your use-case, all the way to creating prototype ready configuration and finally to setup a custom part number for their required configuration before going in production. This tool is intended to provide various use cases where secure elements provide value for customers to reduce risk, increase revenue or reduce cost in their applications. On the top of this page, you will find use-cases in different tabs. In each use-case, we have provided following information for you to easily get started. You will be guided through the following process.

1. USE CASE IDENTIFICATION

From the range of usecases available below choose one that fits closly to your application.Each usecase comes with and example and its respective documentation.

2. TRANSACTION DIAGRAM

Once the use-case is identified, it is important to understand the communication flow between the Host MCU and Secure element for accomplishing the use-case with secure element. Generally each of the use cases would consist in a host sending a “Challenge” and expecting a “Response”. This flow is called the transaction diagram.

In some use-cases, there could be more than one way to achieve the goal based on the secure element features and we will illustrate these paths and explain the pros/cons of each path for the user to select the path that fits best for them.

3. INFER CRYPTOGRAPHIC ASSETS

An asset relates to credentials, token, certificates, key(s) etc.. . A Policy relates to what a memory slot can do such as writing, reading, or both, if the slot is locked or not etc…

As part of the use-case configuration flow, certain cryptographic assets need to be stored inside the secure element and this section will explain the required assets needed along with the security policies needed and the importance of these assets as it pertains to the use-case.

At this stage of the document, it is assumed that the reader has good understanding of the use-case and also the communication protocol between the host MCU and secure element as well as various crypto assets needed in the secure element. It is time to allocate secure element slots for storing necessary cryptographic assets and set the appropriate policy settings. In each of the use-case pages, you will find proposed standard templates. If you want to make any changes to this configuration, you can go to the interactive tool in the  Configurator tab to customize your configuration.

4. SECURE ELEMENT CONFIGURATION

Based on the necessary transactions and associated cryptoagrphic assets and policies, define the Secure Element configuration

Trust&GO and TrustFLEX devices are already configured for a collection of use cases. in case of these device, this step can be skiped.

5. Prototying

Test the configuration and host / Secure Element transactions on the prototyping platform

Cryptoauthlib is C library which supports Microchip CryptoAuth devices. A collection of reference C code examples are available for the most common use cases.

Jupyter Notebook(Python) examples are also available. Notebooks are executable tutorials allowing to prototype the entire set of transactions using the host PC and the actual Secure Element.

6. Secret Exchange process

The Secret Exchange process consists of securely transmitting the secrets, keys and certificate authority to Microchip Hardware Security Modules (HSMs) to allow for volume production

The TrustPlatform Design Suite provides the necessary tools to generate the encrypted secret exchange files.

7. Production Grade Provisioning

Once Secrets, keys and Certificate Authorities are securely loaded in Microchip HSMs, the products can be produced from very low to extremely high volume without the need for any further setup thanks to the extreme scalability of Microchip's provisioning system.

ATECC608A-TNGTLS Device registration

Device claiming process

Trust&GO devices can be automatically claimed by various cloud providers.

Once devices have shipped, you will be able to download the Manifest file from your Microchip Puchasing & Client Services Account. The file can then be uploaded into your cloud service account. Kits, demonstration boards do not ship with a Manifest file. Refer to Trust Platform Manifest file Format documentation for details.

In case of Kits and demonstration boards, running TrustnGO\00_resource_generation\TNGTLS_manifest_file_generation.ipynb notebook will rebuild the manifest for the device on your board.

Upload Manifest to an AWS IoT account

Run TrustnGO\01_aws_connect\notebook\TNGTLS_AWS_connect.ipynb jupyter Notebook to register the devices from your manifest into your AWS IoT account. Step by step instructions are available here: Trust&GO - AWS IoT registration step by step guide

Upload Manifest to an Google IoT account

Run TrustnGO\02_gcp_connect\notebook\TNGTLS_GCP_connect.ipynb jupyter Notebook to register the devices from your manifest into your Google IoT account. Step by step instructions are available here: Trust&GO - Google IoT connect step by step guide
